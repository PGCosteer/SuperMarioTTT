Script

There are three scripts containing one function each that compose the game:

Script that checks if the winning conditions are fullfiled

First of all, three variables are created and values are set. Variable win is a boolean. it turns TRUE when a players wins and acts the proper activities explained later. winnerx and winnero are counters. Every time a player wins, a point is added there.

var win;
var winnerx;
var winnero;
winnerx = 0;
winnero = 0;
win = false;

It's a script with a huge function that is not shown as whole. It contains seventeen IF statements, each for every possible winning combinations, so only one is going to be explained. Everything applies to the other fifteen. Appart of them, there is a last IF statement that is going to be explained separately. It's the DRAW mechanism. So, the function checks the wining conditions by comparing pictures. If certain combinations of pictures are formed in the board, then certain actions are taken. Inside the script's function, the following actions take place:
Checks if a certain combination of pictures the players enter form a winning combination.
If so, the winner is announced, all shells are removed from all blocks (the blank picture is reloaded everywhere), a win point is added to the winner's score (if there is a winner - remember the draw treaty), the turn variable is nullified and the boolean variable winner turns TRUE.

And this is the statement that announces a DRAW. Its a combination of two factors. The idea behind it, is this: "It takes 9 turns to completely fill the board with shells. So, if the ninth turn is reached (if (turn==9) and still no winner is determined (&& win==false)) then take the proper actions." These actions are removing all placed shells from the board (again, the blank picture is reloaded everywhere), the variable turn resets to zero and the message "Draw!" is printed on a specified window. In case someone wins at the ninth turn, then this statement is overlooked and the corresponding winning statement is triggered.

Script that arranges players' turns

This is the main script. It makes the game work by using all previous scripts' functions in one, the ones explained earlier. It's a script with a huge function that is not shown as whole. It contains nine cases, each for every ttt's box, so only one is going to be explained. Everything applies to the other eight. First of all, two variables are created and values are set. The variable player is actually a boolean. It switches between the values one and zero. One for the green player and zero for the red player. Each time a player finishes his/her turn, this variable changes value, letting the other play. Remember the variable turn? After every turn, its value goes up by one. Once a winner is announced or it reaches nine (turns to fill the whole board with shells), or both, it resets to zero, ready to serve for the next match.

var player = 1;
var turn;
turn=0;

Inside the function's switch, the following actions take place:
The case checks who's turn is and if the tile the player pressed is blank. If the pressed tile is blank (the transparent pic is set) then it changes to the player's symbol. After that, the variable turn goes up by one, the variable player changes so the other player can play. Before that, the functions winconditions() and winnerspeaker() check if the winning conditions were fullfiled and print if there was a winner or a draw respectively.
